description: Analyse le code modifié et effectue un commit avec un message détaillé généré par l'IA
---
// turbo-all
1. Vérifie l'état du repo avec `git status`.
2. Inspecte les changements en détail avec `git diff` (utilise un pager ou limite la sortie si trop long).
3. Génère un message de commit descriptif en utilisant le format "Conventional Commits" (ex: `feat: ...`, `fix: ...`, `chore: ...`, `refactor: ...`).
4. Prépare les fichiers avec `git add .`.
5. Valide les fichiers avec `git commit -m "<message généré>"`.
